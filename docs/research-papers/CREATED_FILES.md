## Integration with Documentation

### Creating Research Paper Index in docs/documentation/

**Recommended approach (Option A - Markdown files):**

```bash
mkdir -p docs/documentation/research-papers

# Create index file linking to papers
cat > docs/documentation/research-papers/01-ferroelectric-materials.md << 'DOC'
# Ferroelectric Materials Research

See full paper collection in:
[Research Papers Library](./by-topic/01-ferroelectric-materials/)

Key papers:
- [HZO Superlattice First-Principles](./by-topic/01-ferroelectric-materials/first_principles_hfo2_superlattice_2024.pdf)
- [Preisach Modeling](./by-topic/01-ferroelectric-materials/Preisach_Ferroelectric_Modeling_arXiv.pdf)
DOC
```

**This way:**
- No PDF duplication
- Markdown index files guide readers
- Original PDFs stay in research-papers/
- Links use relative paths

1. Download PDF to `_tools/downloaded/{source}/`
2. Identify correct topic using TOPIC_SUMMARIES.md
3. Move to `by-topic/{NN}-{topic}/` with descriptive name

5. Commit: `git add ... && git commit -m "docs: Add paper on {topic}"`
