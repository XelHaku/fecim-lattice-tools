# Module 1 (Hysteresis) — Automation

This document describes the **automated verification** for Module 1 hysteresis behavior (P–E loops, ISPP, WRD modes) across both physics engines.

## Engines under test

### 1) Preisach (hysteron ensemble)
- **Model:** ensemble of rectangular hysterons with distributed switching thresholds.
- **Strengths:** fast, robust hysteresis loops; convenient for discrete level programming.
- **Typical outputs:** P–E loops, discrete states/levels, switching curves.

### 2) LK / Landau–Khalatnikov (ODE solved with RK4)
- **Model:** Landau free-energy based dynamics: 
  - state variable: polarization **P(t)**
  - driven by electric field **E(t)**
  - integrated via **RK4** time stepping.
- **Strengths:** time-domain dynamics, temperature dependence, transient behavior.
- **Typical outputs:** dynamic P(t), switching transients, physically-parameterized Pr/Ec.

## Waveforms / modes

### P–E loops
- Sweeps the applied field (or voltage) to produce a polarization–electric-field loop.
- Used for extracting **Pr** and **Ec** and for physics regression.

### ISPP target progression (Incremental Step Pulse Programming)
- Applies pulse trains to reach a sequence of target polarization/level setpoints.
- Pass criteria is **convergence**: all target levels reached within a pulse budget.

### WRD modes
- **W**rite / **R**ead / **D**isturb workflows exercised through headless mode tests.
- Ensures target progression and controller state transitions are consistent between engines.

## Validation coverage (what is proven)

The automation suite covers the following claims and failure modes:

1. **Determinism**
   - **Pass:** same seed/config → identical outputs (byte-stable CSV where applicable).
   - Tests: `TestDeterminism_*` (validation).

2. **No NaN/Inf in outputs**
   - **Pass:** no NaN/Inf found in any simulation outputs generated by tests.
   - Tests: `TestDeterminism_NoNaNOrInfInOutputs`, headless verification matrix tests.

3. **Unit consistency (V/m vs MV/cm)**
   - **Pass:** headless runs assert E-field units are internally consistent.
   - Tests: `TestHeadlessLKRun_ISPP5Targets_NoNaNInf_AndEFieldUnitsConsistent`.

4. **Physics regression (golden comparisons)**
   - **Pass:** regression curves match goldens within configured tolerances.
   - Tests: `TestPhysicsRegressionCurves`.
   - Artifacts: golden data in `validation/testdata/`.

5. **ISPP convergence**
   - **Pass:** all target levels reached within pulse budget (no timeouts).
   - Tests: `TestISPPConverges_*` (controller), headless LK convergence matrix / 20-target tests.

6. **Discontinuity classification (0 spurious)**
   - **Pass:** discontinuities are classified as physical only; no spurious steps.
   - Tests: `TestHeadlessISPPContinuityValidation_PreisachVsLK`.

7. **Literature anchoring (experimental datasets, RMSE)**
   - **Pass:** model-vs-literature comparisons remain within RMSE thresholds defined by the validator.
   - Tests: `TestExperimentalDataValidation`.

## Output paths (where artifacts land)

- **Validation goldens / artifacts:** `validation/testdata/`
- **Headless mode test artifacts (when emitted by tests):** under the relevant package `testdata/` folders.
- **CSV logs:** emitted by specific tests/controller runs; the schema stability is enforced by `TestDeterminism_CSVSchemaStable`.

## Failure interpretation (what to do when it fails)

- **Tolerance exceeded (physics regression / literature RMSE):**
  - Indicates a parameter drift, solver change, or golden mismatch.
  - Inspect diffs under `validation/testdata/` and the failing test’s printed deltas.

- **Convergence failure (ISPP):**
  - Targets not reached within pulse budget or a timeout.
  - Usually indicates controller tuning changes, engine behavior changes, or unit scaling issues.

- **Unit mismatch (V/m vs MV/cm):**
  - Typically presents as wildly incorrect switching thresholds or impossible convergence.
  - Re-run headless unit consistency tests and confirm conversions at engine boundaries.

## How to run

Use the unified runner:

```bash
# Fast gate (core proofs)
bash scripts/module1_automation.sh --fast

# Full suite (adds headless matrices + wider package coverage)
bash scripts/module1_automation.sh --full

# Machine-readable summary
bash scripts/module1_automation.sh --fast --json
```

### Modes
- `--fast`: Preisach core + physics regression + controller ISPP convergence + determinism.
- `--full`: adds LK solver tests, M1 package tests, discontinuity validation, experimental-data validation, and headless ISPP runs.
- `--json`: prints a summary JSON object at the end (pass/fail/mode/commit).
